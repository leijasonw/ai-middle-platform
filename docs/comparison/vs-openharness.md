## 四、优劣势对比总结

```text
┌────────────────────────────────────────────────────────────────────────────────────┐
│                      全讯AI中台 vs OpenHarness  优劣势分析                           │

│              │  优势                                                                │
│  OpenHarness ├─────────────────────────────────────────────────────────────────────┤
│              │  • 极轻量 (11K行 vs 500K行)，代码可读性极强                           │
│              │  • MIT许可证，完全开源可商用                                           │
│              │  • 切入点清晰：Harness 基础设施的概念定义非常干净                      │

│              │  • 没有企业级部署方案(Docker/集群)                                     │

│              │  优势                                                                │
│  全讯AI中台   ├─────────────────────────────────────────────────────────────────────┤
│              │  • 多Agent 独立Profile架构，隔离性强                                  │
│              │  • 11个MCP Server，66+专业工具                                        │
│              │  • 109K向量知识库，语义检索                                           │
│              │  • 已连续生产运行1个月，稳定性验证                                     │
│              │  • 完整的产品定价体系 (¥9,800-58,000)                                 │
│              │  • 10个Agent = 10人虚拟团队，直接可感知的价值                          │
│              │  • 飞书原生深度集成                                                   │
│              ├─────────────────────────────────────────────────────────────────────┤
│              │  劣势                                                                │
│              │  • 依赖 Hermes 框架，定制灵活性受限                                    │
│              │  • 没有 Dry-run 预览机制                                              │
```

## 六、我们可以从 OpenHarness 学什么

```text
┌────────────────────────────────────────────────────────────────────────────────────┐
│                    全讯AI中台可以借鉴的能力                                           │
├────────────────────────────────────────────────────────────────────────────────────┤
│                                                                                    │
│  1.  🚀 Dry-run 安全预览机制                                                       │
│      ─────────────────────────                                                     │
│      OpenHarness 的 oh --dry-run 可以在不执行模型/工具的前提下预览Agent会做什么     │
│      → 我们可以在Gateway层加一个"预览模式"：用户发消息前先看看Agent会调什么工具       │
│      → 低成本高安全感知的功能，客户很吃这一套                                        │
│                                                                                    │
│  2.  🛡️ 三级权限模式 + 路径规则                                                     │
│      ────────────────────────────────                                               │
│      OpenHarness 的 Default/Auto/Plan 三级权限                                      │
│      → 我们的审批smart模式可以借鉴，扩展为更细粒度的Agent权限控制                     │
│      → 企业版客户特别需要这个                                                       │
│                                                                                    │
│  3.  🎯 Skills 系统 (.md 按需加载)                                                  │
│      ──────────────────────────────────                                               │
│      OpenHarness 用纯 Markdown 文件定义领域知识                                     │
│      → 我们的 Agency 角色 188个可以借鉴这种"轻量加载"方式                             │
│      → 降低Agent技能的定义门槛                                                      │
│                                                                                    │
│  4.  📦 极简安装体验                                                                │
│      ────────────────                                                               │
│      OpenHarness: curl | bash 一键安装 + uv sync                                   │
│      → 我们的Docker Compose部署脚本可以借鉴这种"一条命令搞定"的体验理念               │
│                                                                                    │
│  5.  🧩 插件生态兼容 Claude Code                                                    │
│      ──────────────────────────────────                                               │
│      OpenHarness 兼容 anthropics/skills 格式                                       │
│      → 我们的Profiles也可以考虑兼容Hermes社区标准                                    │
│                                                                                    │
└────────────────────────────────────────────────────────────────────────────────────┘
```
