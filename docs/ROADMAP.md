## 三、MCP 工厂：核心创新设计

这是全讯中台区别于 OpenHarness 和其他框架的最核心模块。

### 3.1 设计理念

```text
┌──────────────────────────────────────────────────────────────────────────┐
│  OpenHarness 的做法:                                                      │
│    MCP 客户端 → 手动配置 host/port → 手动安装依赖 → 手动配token           │
│    问题: 每加一个MCP都要5-10分钟手动操作，不适合企业批量部署               │
│                                                                          │
│  我们的做法 — MCP Factory:                                                │
│    qxmcp scan → 自动发现GitHub上符合MCP协议的工具                         │
│    qxmcp install → 一键安装 (环境/依赖/token/权限 全自动)                  │
│    qxmcp verify → 沙箱验收 (功能测试+安全扫描+兼容性检查)                  │
│    qxmcp bind   → 绑定到指定Agent Profile (~2秒完成)                       │
│                                                                          │
│  效果:                                                                    │
│    10分钟手动操作 → 2秒自动化                                              │
│    手误风险消除 → 沙箱验收保障                                             │
│    单Agent专用 → MCP全Agent共享                                            │
└──────────────────────────────────────────────────────────────────────────┘
```

### 4.3 Hermes 上游更新策略

```text
┌──────────────────────────────────────────────────────────────────────────┐
│                    Hermes Update Strategy                                 │
│                                                                          │
│  hermes-core/ → git submodule → nousresearch/hermes                      │
│                                                                          │
│  # 检查上游更新                                                          │
│  bash hermes-core/upgrade.sh check                                        │
│      输出: Current: v0.18.2 | Latest: v0.19.0                            │
│                                                                          │
│  # 执行更新                                                              │
│  bash hermes-core/upgrade.sh apply                                        │
│      1. git pull origin main (submodule)                                 │
│      2. 检查Breaking Changes (CHANGELOG.md)                             │
│      3. 运行兼容性测试 (profiles/* 验证)                                 │
│      4. 如果通过 → 提示重启Gateway                                      │
│      5. 如果失败 → 提示需要手动调整的Profile列表                         │
│                                                                          │
│  优势:                                                                   │
│    · 始终使用最新Hermes，不固化版本                                      │
│    · Breaking Changes自动检测(而非惊喜)                                  │
│    · 一键回滚到上一个稳定版                                               │
│                                                                          │
└──────────────────────────────────────────────────────────────────────────┘
```
